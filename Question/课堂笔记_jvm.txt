		各个分代垃圾回收的名称
			年轻代: minor  年轻代对象的特点: 对象数量多 对象的存活时间短 需要高频率的进行回收
			老年代: major  老年代对象的特点: 对象数量少 对象的存活时间长 无需高频率的进行回收
			整堆  : Full 
		jvm堆内存调优入门
			堆参数的特点  以-开头 分类如下
			-version: 系统版本参数
			-X      : 一个X的参数比较稳定 不会随着jdk版本的迭代发生显著的变化
			-XX     : 两个X的参数不稳定,所对应的配置项可能会随着jdk版本的迭代而取消
		-X
			-Xms: 默认初始使用的堆内存大小 1/64
			-Xmx: 最大堆大小 1/4
			-Xmn: 新生区堆大小
		-XX
			-XX:+PrintGCDetails  输出详细的GC处理日志

				jdk17取而代之的是-Xlog
		案例:

	11,垃圾回收算法
		Stop-the-World 简称STW 垃圾线程在执行的时候 整个业务线程是要停止的
		复制算法: 主要用于新生区的垃圾回收  新生区中的对象存活低
				 优点: 快 效率高
				 缺点: 浪费一半的内存空间 s0和s1是交替有值
		标记清除: 主要用于养老区的垃圾回收 因为老年代对象过大如果直接复制开销太大,所有就地标记
				 优点: 节省空间
				 缺点: 先标记后清除时间长   会产生内存碎片
		标记压缩: 也叫作标记整理 主要用于养老区的垃圾回收  因为老年代对象过大如果直接复制开销太大,所有就地标记
				 优点: 节省空间
				 缺点: 先标记后清除时间长	
		分代收集
			新生区: 使用复制算法
			养老区: 使用标记清除或者标记压缩再或者二者混合

		1,分类 
			串行: 如果年轻代使用Serial那么养老区要使用Serial Old GC 成对出现
				Serial: 回收年轻代  采用复制算法
				Serial Old GC: 养老区  采用标记压缩算法
			并行(JDK8默认)如果年轻代使用Parallel那么养老区要使用Parallel Old GC 成对出现
				Parallel : 回收年轻代  采用复制算法
				Parallel Old GC: 养老区  采用标记压缩算法
			并发
				CMS Concurrent-mark-sweep   养老区  采用标记清除算法--致命缺点内存碎片
					与CMS搭配进行年轻代垃圾回收的是ParNew-->全称Parallel new

		并发垃圾回收器: --- 低延迟 低STW
			鼻祖: CMS
			CMS老年代并发回收器的回收过程
			标记
				初始标记: 与GCroot直接关联的对象 不怎么耗时
				并发标记: 与GCroot间接关联的对象 耗时      该阶段是与用户线程并发执行
				重新标记: 并发执行过程中用户线程产生的垃圾对象
			清理
				并发清理 耗时      该阶段是与用户线程并发执行
			缺点:
				由于不需要移动存活对象会产生内存碎片
		G1: 并发垃圾回收器 ---吞吐量  多核cpu 和 大内存(>=8G) 200ms
			The Garbage-First (G1) garbage collector is targeted for multiprocessor machines with a large amount of memory
			1.堆划分是按照Region进行的
				红色: 伊甸园
				红色带S: 幸存区
				浅蓝色: 老年代
				大浅蓝色: 大对象
			2,Remembered  Set 和 Card Table
				作用: 需要知道哪些对象有跨 Region 的引用
				Remembered  Set 简称 RSet 记忆集   老板
				Card Table  简称: 卡表             秘书
			3,G1垃圾回收过程
				初始标记: 与GCroot直接关联的对象 不怎么耗时
				并发标记: 与GCroot间接关联的对象 耗时      该阶段是与用户线程并发执行无需关系
				再标记:  并发执行过程中用户线程产生的垃圾对象 利用写屏障避免漏标问题
				清理:    清点出有存活对象的分区和没有存活对象的分区 即扫描Rset
				复制:    由于底层采用标记整理算法 会涉及到对象的复制
			4, 三色标记法:
				作用: G1在并发标记阶段判断对象存活性的核心算法
				白色:垃圾对象
				灰色:待扫描的存活对象
				黑色:存活对象
			5, 三标标记法在极端情况下可能会产生的问题
				漏标: 把已标记的对象误标为垃圾    无法接受,必须解决
				多标: 把已经不存活的对象误标为存活 可以接受
			6, 关于漏标问题
				1,采用SATB解决漏标问题的
				2,原理:
					在并发标记阶段(应用线程和GC线程并发执行),当应用程序断开,会出发一个写屏障
					这个写屏障会把对象之前的旧引用值保存在一个特殊的队列中
					在重新标记阶段会扫描这个特殊队列 会将白色的对象强制转换为黑色对象

		ZGC: 并发垃圾回收器 降低RT 延迟低于10ms (了解)

	1,jdk jre jvm 分别是什么 他们之间关系是什么
		JDK: java development kit java开发工具包
		jre: java runtime environment java运行环境
		jvm : java virtual machine java虚拟机
		关系: jdk包含jre jre包含jvm
	2,JVM结构
		这个JVM包含三个子系统 分别是类加载子系统 运行时数据区子系统 执行引擎子系统
		类加载子系统 主要负责将字节码文件加载到运行时数据区子系统
		运行时数据区子系统分为线程公有和线程私有部分
			线程公有不分包含java堆和方法区
			线程私有部分包含java栈 本地方法栈 和 程序计数器
		执行引擎子系统 负责程序的执行 和垃圾回收相关的操作
