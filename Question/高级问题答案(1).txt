JUC
 1.问：线程创建方式
    答：第一种：继承Thread  重写 run 方法  第二种：实现 Runable接口 重写run方法  第三种：实现Callable接口 重写call方法  第四种：线程池。
 2.问：Runable和Callable创建线程时候的区别
    答：Runable重写run方法 且没有返回值，Callable重写call方法 且有返回值。
 3.问：线程生命周期(状态)
    答：五种，创建--就绪--阻塞（睡眠或等待）--运行--销毁    //  六种，创建--就绪--睡眠--等待--运行--销毁 
 4.问：线程安全问题产生的原因
    答：1多线程操作共享资源2,操作共享资源的代码有多行
 5.问：死锁产生的原因都有哪些,以及如何避免死锁
    答：互斥，占有且等待，不可剥夺，循环等待 。
	    避免：设置超时时间，线程按业务顺序拿锁，
 6.问：说说synchronized底层原理
    答：是java内置的互斥锁，底层基于 对象头 markwork和 监视器锁实现（monitor）实现，有四种状态，无所，偏向锁，轻量级锁，重量级锁。偏向锁适合单线程无竞争，轻量级锁适合多线程无激烈竞争，才用自旋抢锁，竞争激烈的适合为重量级锁，线程进入阻塞队列。
 7.问：synchronized和lock区别 
    答：synchronized：c++实现，无需手动释放，功能较少，没有竞争性能更好。
            lock                ：jdk提供，需要手动释放，功能丰富，竞争激烈性能更好。   
 8.问：死锁产生的原因都有哪些以及如何检测死锁
    答：多个线程长时间处于阻塞状态。
 9.问：ReentrantLock底层原理(相当于就是问AQS底层原理)
    答：ReentrantLock底层就是通过修改aqs的state状态逻辑完成的，
	    aqs底层原理主要就是一套抢锁和唤醒的规则，以及state状态变量和clh双向阻塞队列。
            state通过0和1来判断是否含有锁，当state为0的时候证明无锁，1则为有锁，实现重入是通过state累加，当前线程再次访问锁的时候，state加1，释放则减1,0彻底释放。
            clh是一个双向链表，没有抢到锁的线程，被aqs扔入队列中，不必自旋抢锁，state为0的时候，会唤醒队列中的线程。
 10.问：谈谈你对JMM的理解
       答：java memory model    对jvm中对于线程安全保障的一种规范（接口）  ，屏蔽操作系统和cpu不同带来的影响，一套抽象的规范。使java可以一处编译到处运行。 
              原理就是将内存分为两个区域，一个主内车，一个工作内存每个线程私有的。
              解决三大特性：原子性：sychronized，lock 保证原子性，多线程并发操作不可分割
                                        可见性：mesi协议 保证可见性，硬件的一致性协议，主内存和线程的工作内存之间的一致性。
                                        有序性：通过内存屏障  loadload屏障 storestore屏障 loadstort屏障 storeload 屏障。可以通过synchronized 和 volatile 禁止指令重排
 11.问：谈谈你对Volatile的理解
       答：禁止指令重排 
               jmm规模的一种实现，一种轻量级的同步机制， 不保证原子性 ，保证 可见性，有序性
               只修饰变量不修饰方法，与synchronized不同，synchronized不修饰变量，修饰方法和代码块。
 12.问：Volatile不保证原子性通过什么可以保证原子性
       答：解决volatile不保证原子性的缺陷，可以使用原子类，也就是CAS。
 13.问：原子类底层CAS的原理
       答：乐观无锁机制，底层基于 CPU cmpxchg 原子指令。通过属性的偏移量修改。
              步骤 一.获取要修改属性的偏移量，二.通过Unsafe类提供的compareAndSwapXXX（修改对象，偏移量，旧值，新值）三.匹配则替换，否则不修改也不报错。
 14.问：如何理解并解决ABA问题
       答：理解：线程1执行过程中，线程2对线程1需要操作的数据进行操作并且还原，线程1只对数据进行校验并未发现。
               解决：版本号  使用带版本号的原子类  AtomicStampedReference(原值，原版本号)   
 15.问：异步编排常用的API有哪些?默认使用的线程池名称是什么?
       答：runAsync/supplyAsync  创建异步任务，   thenRun/thenApply   回调
               forkJoinPool 默认线程池
 16.问：谈谈你对线程池的理解
      答：池化技术，资源复用（降低创建和销毁的成本）。控制最大并发数，管理线程（统一的调优和监控）。性能提升（任务不用等待线程创建）
 17.问：线程池七大参数 运行流程 拒绝策略有几种(5种)
      答：核心线程数--最大线程数--最大线程过期时间--最大线程过期时间单位--阻塞队列--线程工厂（默认）--拒绝策略（默认）
              工作流程：先用核心线程 不足则将任务存入阻塞队列，阻塞队列满后创建救急线程，若是任务大于最大线程数则触发拒绝策略。任务执行结束，救急线程会被销毁，核心线程则放入线程池。
              拒绝策略有五种  AbortPolicy（拒绝任务并抛出异常） DiscardPolicy（拒绝任务不抛异常）DiscardOldestPolicy（拒绝等待时间最久的任务，重试其他任务）CallerRunsPolicy（调用主程序执行任 务，不拒绝任务）    自定义拒绝策略
JVM
	1.问：jvm组成结构
           答：jvm中包含三个子系统，分别是类加载子系统，运行时数据去子系统，执行引擎系统
                   类加载子系统：主要负责将字节码文件加载到运行时数据区子系统
		   运行时数据去子系统：分为线程公有和线程私有部分，公有部分包含java堆和方法区  私有部分包含java栈 本地方法栈 和 程序计数器。
                   执行引擎系统：负责程序的执行 和 垃圾回收相关的操作
	2.问：堆内存结构划分
	   答：jdk7之前：新生区，养老区，永久代
                  jdk7之后：新生区，养老区，元空间
                  新生区：伊甸园区
				 幸存0区  == s0  ===from区
				 幸存1区  == s1  ===to 区
	3.问：双亲委派流程是什么?他的作用是什么	
           答：流程 由应用类加载器发起，将加载任务上交父类扩展类加载器，再有扩展类加载器上交父类根加载器， 根加载器到rt.jar中加载，若没有找到，则下发到扩展类加载器去加载其他jar包，若没有找到则	           由应用类加载器到第三方jar中加载，若没有找到则抛出异常
                   作用 保证java核心类库源码安全性 --- 沙箱安全机制    避免出现重复的类信息。
	4.问：对象的引用方式有几种		
	   答：强引用：User user=new User();	永不回收																
	           软引用： SoftReference                     内存不足回收
	  	   弱引用：WeakReference                    gc就回收
                   虚引用：PhantomReference              跟没有任何引用是一样的
	5.问：如何判断一个对象为垃圾对象
            答：1）引用计数法（废弃）
		    2）可达性分析
	6.问：垃圾回收算法有哪些?各自的优缺点
	    答：复制算法：主要用于新生区    新生区对象存活低，优点：快，效率高。缺点：浪费一半的内存空间，s0和s1交替有值
                    标记清除：主要用于老年区    老年区对象比较大，直接复制消耗高  优点：节省空间。缺点：产生碎片空间
                    标记压缩：主要用于老年区    优点：节省空间 缺点：时间长
                    分代收集（新生区用复制算法，养老区标记清除或标记压缩）
	7.问：垃圾回收器的分类(串行 并行 并发)
           答：串行   Serial    +   Serial Old 
		   并行   Parallel  +   Parallel Old 
		   并发    ParNew + CMS(老年代使用标记清除 产生内存碎片   淘汰)   //  G1   // ZGC 
	8.问：CMS垃圾回收器回收算法是什么? 缺点是什么?全称是什么
	   答：使用 标记--清除 算法，只回收老年代。  
                   缺点：由于不需要移动存活对象会产生内存碎片 
		   全称：Concurrent Mark Sweep
	9.问：G1回收器(包含介绍 记忆集Rset和卡表 三色标记算法以及漏标问题如何解决)
           答： Garbage First 垃圾优先 ，不分新生，老年代，划分多个大小相等的区域 （region）默认2048个 不同区域可以同时回收
		    优先回收垃圾最多，回收时间较短的region，可以使用 -XX：MaxGCPauseMillis设置最长停顿时间。
		    适用于 多核cpu 和 大内存（>=8g)    底层使用的是标记整理算法
	            垃圾回收过程 ： 初始标记，并发标记，再标记，清理，复制。
		    记忆集Rset ： 记录跨region对象
 		    卡表：
		     三色标记法：并发标记阶段 用来判断对象是否存活的算法    
						白色：垃圾对象  灰色：带扫描的存活对象  黑色：存活对象
		     漏标：白色对象被黑色对象重新引用，但是白色对象被当做垃圾清除 
				 解决办法：采用SATB解决
	10.问：G1垃圾回收器的回收过程 
	      答：五个阶段 初始标记--并发标记--再标记--清理--复制
		     初始标记：与GCroot直接关联的对象，不怎么耗时
		     并发标记：与GCroot间接关联的对象，耗时
		     再标记：   并发执行过程中用户线程产生的垃圾对象      利用写屏障避免漏标问题
		      清理：      清点出有存活对象的分区和没有存活对象的分区   扫描 rset（记忆集）
		      复制：    由于底层采用的是标记清理算法，会涉及到对象的复制。
MySQL
	1.问：内连接外链接区别
	   答：内连接取两表交集，外连接取主表全部数据和另一张表匹配上的数据
	2.问：如何优化SQL语句说说思路
	   答：根据explain 执行计划 增加设置索引，避免索引失效的情况，关联查询小表驱动大表 
	3.问：B+Tree和B-Tree区别																						
           答：Btree中每个节点都包含主键，下一个节点的指针以及所有数据 。 b+tree中非叶子节点 只包含主键和下一个节点的指针。
		   b+tree 的叶子节点做了一个双向指针，查询次数相对固定。而且有序
	4.问：树高为什么是3是最合适的?
           答：3层会产生两次io，对用户体验来说相对平缓，如果树高四层，会产生三次io，会有明显感觉，两层树高查询数据太少不适合。
	5.问：什么是回表
           答：非聚集索引，查询数据的时候，先去查询主键，再用聚集索引去查询数据。
	6.问：聚簇索引和非聚簇索引区别?主键一定是聚簇索引吗?
	   答：在innoDB中 主键上加的索引叫做聚簇索引    在MyISAM 中不是（不太懂）
	7.问：说说什么是索引下推
           答：using index Condition  （icp）   减少回表次数    将扫描数据的操作下推到数据的存储引擎层
	8.问：MySQL事务的特性 不考虑事务隔离性可能会产生哪些问题(具体讲讲脏读 不可重复读 幻读的概念)
	   答：脏读：一个事物对某数据进行修改，尚未提交数据库时，另一个事物读取该数据，这时候读到的数据就是脏数据
		  不可重复读：针对修改数据，一个事物对某数据进行多次读取，另一个事物在此期间对数据进行修改，使第一个事物多次读取结果不一致。
                  幻读：针对新增或者删除操作，一个事物对多条数据进行读取，另一个事物在此期间新增或者删除数据，使第一个事物读取数据总数发生变化。
	9.问：事务特性分别是由哪些日志保证的? 
	   答：undo log  ， redo log ，mvcc 以及锁
	10.问：MySQL中的事务隔离级别都有哪些?分别可以解决哪些问题
	      答：读取未提交：运行读取尚未提交的数据变更，可能导致 脏读幻读不可重复读
		      读已提交：运行读取并发事物已提交的数据，不会脏读，可能幻读，不可重复读
		      可重复读：对同一字段多次读取一直，不会脏读，不可重复，可能幻读
		      可串行化：最高隔离，性能差
	11.问：ACID分别是有哪些日志保证的?
	      答：A=原子性（一个事物，要么都完成，要么都不完成）（undo log），C=一致性（一个事物执行前后数据库状态一致）（MVCC undo log），
		       I=隔离性（并发环境中，一个事物所做的修改必须与其他事物隔离，事物查看数据更新时，数据要么是未被修改要么是已被修改，，不可以处于中间状态）（锁），
                     D=持久性（事物结束，该事物对数据库的更新就必须保持下来）（redo log）
	12.问：讲讲MVCC流程
	      答：多版本并发控制，并发读写不加锁 
                      三个核心，默认字段 事务id和回滚指针，undolog ，readview
		      数据更新的时候，原数据不会删除，而是保存在undolog中，新数据的回滚指针指向旧数据，旧数据的回滚指针指向更上一版的数据，
		      另一个事物开启查询的时候，会生成readview快照，根据可见规则去查询当前事物可以查询的数据。
		      其中，读未提交是没次查询都会生成一次快照，可重复读是只在第一次查询的时候生成快照以此来解决不可重复。
	13.问：讲讲分库分表的思路(主要说水平拆分和垂直拆分的区别)
	      答：水平：将一张大表拆分成两张一模一样的小表，这两个小表中的字段完全一致。
    		      垂直：将一张大表拆分成两张子表，分别包含不同的字段，两张子表通过逻辑字段相连
	14.问：MySQL主从复制原理是什么
              答：slave 会从 master 读取 binlog日志 来进行数据同步 
	             1.master将数据改变记录到二进制日志 binlog 中
                     2.slave创建一个io现场请求master中的binlog
		     3.master创建一个log dump 现场，，发送binlog内容， 读取binlog的时候会加锁，读取发送给slave的时候会解锁。
      		     4.io现场接受log dump 线程发来的更新后，会保存到中继日志，relay log 中
		     5.slave的sql现场，读取中继日志 ，并解析 跟master中数据一致
